Features

Visuals, chat assists, and bot integration.

Visuals

  • Boost pad timers with reactivation countdowns
  • Car ESP with per‑car accurate hitboxes
  • Controller overlay (compare bot vs human inputs in one view)

Chat

  • AI auto‑responses in quick‑chat style
  • Defensive replies to accusations or toxicity (toggleable)

Bot

  • Integrated Nexto bot
  • Splitscreen support (multiple bots in one instance)
  • Auto skip replay for all local players
  • Auto forfeit for local players
  • New Auto Queue (let the bot play AFK)
  • New Optional Discord Webhook for game logs

SDK / Dev

Vutrium exposes an internal SDK for reading game state (players, ball, boosts) and writing inputs in process, aiming to stay compatible with the game’s physics and control model.

  • DLL bridge for real‑time memory access
  • Data frames: positions, velocities, boost states, events
  • Client control path for inputs at high frequency

How it Works

Real‑time, bidirectional pipeline between the game and the client.

Data Flow Architecture

DLL injection sets up an in‑process SDK inside Rocket League. Game state streams to the client (AI / Nexto / tools), which returns control inputs at high frequency.

Vutrium is for educational/experimental use. Using tools that modify game memory or automate inputs in online modes may violate the game’s Terms of Service and can result in bans. Proceed at your own risk.

Usage & Requirements

Fast setup with a minimal UI for configuration.

How to run

  • Run Vutrium.exe — it will inject the SDK and initialize.
  • Open the in‑game GUI with Insert.
  • Bot toggle key: F1.
  • Optional: set a Discord webhook URL for game logs.

Requirements

  • Windows 10/11 (64‑bit)
  • Rocket League on PC
  • CPU: Dual‑core 2.0 GHz or higher • RAM: 4 GB • Storage: 1 GB
  • GPU: Integrated is fine (Nexto runs on CPU)
  • Python 3.11 may be required

These are minimums for Vutrium, not for Rocket League itself.
